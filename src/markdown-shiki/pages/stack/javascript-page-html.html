    <!--  -----  Features  -----  -->
    <section class="js-page__features">

        <h3 class="js-page__features-title">Características Principales</h3>

        <div class="js-page__features-grid">

            <article class="js-page__feature">
                <span class="js-page__feature-icon" aria-hidden="true">📦</span>
                <h4 class="js-page__feature-title">Variables y Scope</h4>
                <p class="js-page__feature-text">
                    <code>const</code> y <code>let</code> reemplazan a <code>var</code>,
                    ofreciendo scope de bloque y evitando los problemas de hoisting.
                    Las closures permiten encapsular estado de forma elegante.
                </p>
            </article>

            <article class="js-page__feature">
                <span class="js-page__feature-icon" aria-hidden="true">➡️</span>
                <h4 class="js-page__feature-title">Arrow Functions</h4>
                <p class="js-page__feature-text">
                    La sintaxis <code>() =&gt; {}</code> proporciona funciones más concisas
                    y mantiene el contexto léxico de <code>this</code>,
                    eliminando uno de los puntos de confusión más comunes del lenguaje.
                </p>
            </article>

            <article class="js-page__feature">
                <span class="js-page__feature-icon" aria-hidden="true">🔗</span>
                <h4 class="js-page__feature-title">Promesas y Async/Await</h4>
                <p class="js-page__feature-text">
                    Las <code>Promise</code> y la sintaxis <code>async/await</code>
                    permiten gestionar operaciones asíncronas como llamadas a APIs
                    de forma legible y sin el temido "callback hell".
                </p>
            </article>

            <article class="js-page__feature">
                <span class="js-page__feature-icon" aria-hidden="true">🧩</span>
                <h4 class="js-page__feature-title">Módulos ES (ESM)</h4>
                <p class="js-page__feature-text">
                    Con <code>import</code> y <code>export</code>, JavaScript dispone de
                    un sistema de módulos nativo que organiza el código en archivos
                    independientes, reutilizables y con dependencias explícitas.
                </p>
            </article>

            <article class="js-page__feature">
                <span class="js-page__feature-icon" aria-hidden="true">🏛️</span>
                <h4 class="js-page__feature-title">Clases y Herencia</h4>
                <p class="js-page__feature-text">
                    La sintaxis <code>class</code> proporciona una forma más clara de
                    definir objetos y herencia basada en prototipos, con constructores,
                    métodos, getters/setters y herencia mediante <code>extends</code>.
                </p>
            </article>

            <article class="js-page__feature">
                <span class="js-page__feature-icon" aria-hidden="true">✂️</span>
                <h4 class="js-page__feature-title">Desestructuración y Spread</h4>
                <p class="js-page__feature-text">
                    La desestructuración de arrays y objetos, el operador spread (<code>...</code>)
                    y los parámetros rest simplifican el trabajo con datos complejos
                    y reducen el código repetitivo.
                </p>
            </article>

        </div>
